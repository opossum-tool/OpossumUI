# AGENTS.md

## Project overview

OpossumUI is an Electron app (TypeScript + React + Redux) for auditing open-source license compliance data from `.opossum` files. SQLite backend exposed to frontend via a typed API through `backendClient`.

## Architecture

- **`src/ElectronBackend/`** — Electron main process: app entry (`app.ts`), preload (`preload.ts`), DB worker (`dbProcess/`), API commands (`api/`), SQLite via Kysely (`db/`)
- **`src/Frontend/`** — React UI: components (`Components/`), Redux state (`state/`), utilities (`util/`)
- **`src/shared/`** — Types, constants, IPC channels shared between frontend and backend
- **Two separate tsconfig projects**: root tsconfig (Frontend + shared, excludes ElectronBackend) and `src/ElectronBackend/tsconfig.json`. Both must pass.

## Critical quirks

- **Node version pinned** in `.nvmrc` (see file for exact version). Yarn 4 (Berry).
- **`better-sqlite3` dual build**: Installed twice — once for Node.js (used by Vitest/scripts), once for Electron (`better-sqlite3-electron`, used at runtime). Vite aliases `better-sqlite3` → `better-sqlite3-electron` for the Electron build. If you see ABI mismatch errors: `yarn rebuild:electron`.
- **`src/ElectronBackend/db/generated/`** is auto-generated by `yarn db:generate`. Never edit these files manually. Run `yarn db:generate` before typecheck whenever backend DB schema changes; `typecheck` script does this automatically.
- **REUSE copyright headers** (SPDX) are required on every source file. `yarn copyright-lint-check` enforces this. Pre-commit hook also checks it.
- **Exact dependency versions** are enforced: no `^` or `~` allowed in `package.json`. `yarn check-exact-versions` checks this (runs on commit via lint-staged and in CI).

## Commit conventions

- **Conventional commits** enforced via commitlint: `type(scope)?: description` where type is one of: `build`, `chore`, `ci`, `docs`, `feat`, `fix`, `perf`, `refactor`, `revert`, `style`, `test`.
- **DCO sign-off** required on every commit (`git commit -s`).

## Pre-commit hook

Husky + lint-staged runs on every commit: copyright check, knip, prettier, eslint, and tsc (with `db:generate` for ElectronBackend file changes). This means a commit may fail on checks beyond just the files you changed.

## ESLint rules that bite

- **No `@faker-js/faker`** direct imports — use the shared faker utility instead.
- **No `react-hot-toast`** direct imports — use toast from `Toaster`.
- **No default React import or namespace import** (`import React` / `import * as React`) — new JSX transform handles this.
- **No `styled()` inside functions** — must be at module scope or in `.style.ts` files.
- **`@typescript-eslint/no-explicit-any`** is error — no `any`.
- **`@typescript-eslint/consistent-type-imports`** with `inline-type-imports` — use `import { type Foo }` not `import type { Foo }`.
- **`@typescript-eslint/no-magic-numbers`** — banned except -1, 0, 1, 2, 100 (relaxed in test files).
- **Filename convention** (`check-file/filename-naming-convention`): kebab-case with optional `.d`, `.test`, `.style`, `.util`, `.util.test` suffixes before the extension.

## Vitest setup

Three vitest projects with different environments:

- **FE** (happy-dom): `src/Frontend/**/__test{s,}__/**/*.test.{ts,tsx}`
- **BE** (node): `src/ElectronBackend/**/__test{s,}__/**/*.test.{ts,tsx}`
- **SH** (node): `src/shared/**/__test{s,}__/**/*.test.{ts,tsx}`

Test directories can be `__tests__` or `__test__`.

## E2E testing

- Playwright. `yarn test:e2e` auto-starts the dev server; `yarn test:e2e:ci` tests against a built release.
- `WORKERS` env var controls parallelism (default: 1). Set in `.env` (see `.env.sample`).
- `debug()` fixture pauses a test so you can open the generated `.opossum` file in the running app.
- Artifacts (traces, .opossum files) land in `src/e2e-tests/artifacts/`.

## Style

Prettier: single quotes, semicolons, trailing commas, import sorting (third-party then relative, separated by blank line).
